# 南宁通信段劳动用工工时、工分统计管理系统需求分析

## 一、系统建设背景

目前南宁通信段劳动用工工时、工分统计主要依赖 Excel 表格进行填报和汇总。Excel 表中包含多个工作表，例如：

```text
汇总表
车间
A工区
B工区
C工区
……
H工区
填报说明
```

每个工区或车间本级需要填写人员每日的工时、工分数据，最后由车间和段级进行汇总。现有需求文档中已明确当前系统需要实现工区填报、车间审核、段级汇总、Excel 导出、权限控制和数据追溯等核心能力。

现有 Excel 填报方式存在以下问题：

1. 表头复杂，包含大量多级合并表头。
2. 工区、车间、段级之间的数据权限难以控制。
3. Excel 文件反复传递，容易出现填错、漏填、覆盖、版本混乱等问题。
4. 汇总需要人工处理，效率较低。
5. 数据修改过程缺少记录，后期难以追溯。
6. 工时、工分、备注、培训班名称等字段类型混合，容易混乱。
7. 人员调动后，历史数据和当前人员归属容易混乱。
8. 各车间可能有多个管理员或领导，需要多人共同管理本车间数据。
9. 需要对工时、工分数据进行统计分析。
10. 工区导出的 Excel 表格需要尽量与原模板保持一致，方便继续存档、上报或打印。

因此，需要建设一个线上管理系统，实现劳动用工工时、工分数据的线上填报、审核、汇总、统计分析、Excel 导出和全过程追溯。

------

## 二、系统定位

系统名称建议为：

```text
南宁通信段劳动用工工时、工分统计管理系统
```

系统定位为：

> 面向南宁通信段内部使用的劳动用工工时、工分填报、审核、汇总、统计分析和 Excel 导出管理系统。

系统不面向公众开放，不需要用户自主注册。所有账号由段级统一创建、下发和管理。

------

## 三、系统建设目标

系统建设目标如下：

1. 实现工区线上填报工时、工分数据。
2. 实现车间对下属工区数据进行审核、退回和汇总。
3. 实现段级对所有车间、工区数据进行查看、汇总、统计和锁定。
4. 实现段级、车间、工区三级数据权限控制。
5. 实现用工项目多级字典管理，适配 Excel 中复杂多级表头。
6. 实现工时、工分、备注、培训班名称等不同类型数据的分类存储。
7. 实现人员信息管理和人员调动处理。
8. 支持一个车间配置多个车间管理员或领导账号。
9. 支持统计分析功能，方便按车间、工区、人员、项目、月份等维度查看数据。
10. 支持 Excel 导出，工区导出表格需要尽量与原模板一致。
11. 记录提交、审核、退回、修改、导出、锁定等操作日志，方便后期追溯。

------

## 四、组织架构需求

系统采用三级组织结构：

```text
段级
 └── 车间
      ├── 车间本级
      ├── 具体工区1
      ├── 具体工区2
      ├── 具体工区3
      └── ...
```

业务规则如下：

1. 段级为最高级。
2. 段级下面有 15 个车间。
3. 每个车间下面有 3 到 7 个具体工区。
4. 工区名称使用真实工区名称，例如：
   - 梧州通信工区
   - 道州通信工区
   - 贺州通信工区
   - 岑溪通信工区
5. 不同车间下属的工区数量可以不同。
7. 系统应支持段级管理员维护车间与具体工区的对应关系。

例如：

```text
贺州通信车间
 ├── 梧州通信工区
 ├── 道州通信工区
 ├── 贺州通信工区
 └── 岑溪通信工区
```

组织架构主要用于：

1. 控制账号的数据权限范围。
2. 确定人员所属车间和工区。
3. 确定工区填报数据归属。
4. 支撑车间汇总和段级汇总。
5. 支撑 Excel 导出时的数据归属和模板工作表映射。

段级管理员是系统业务最高权限账号。

### 1. 可使用功能

段级管理员可以使用以下功能：

1. 登录系统。
2. 修改本人密码。
3. 查看段级首页看板。
4. 管理账号。
5. 管理组织架构。
6. 管理人员信息。
7. 管理人员调动。
8. 管理用工项目字典。
9. 创建和管理填报月份。
10. 查看所有车间数据。
11. 查看所有工区数据。
12. 修改所有车间和工区数据。
13. 查看段级汇总。
14. 查看统计分析。
15. 导出 Excel工区表。
17. 导出车间汇总表。
18. 导出段级汇总表。
19. 锁定和解锁月份。
20. 查看审核记录。
21. 查看操作日志。

### 2. 数据范围

段级管理员可以查看和管理：

```text
所有车间
所有工区
所有人员
所有月份
所有填报数据
所有审核记录
所有操作日志
```

### 3. 人员调动权限

段级管理员可以对人员所属组织进行调整。

例如：

```text
张三原属于梧州通信工区
段级管理员可以将张三调整到贺州通信工区
```

人员调动要求：

1. 支持修改人员所属车间。
2. 支持修改人员所属工区。
3. 支持记录调动前车间、调动前工区。
4. 支持记录调动后车间、调动后工区。
5. 支持记录调动原因。
6. 支持记录操作人和操作时间。
7. 调动后，新填报数据按新的所属组织归属。
8. 历史月份数据原则上保留原所属组织，避免历史数据被错误覆盖。

可以先实现人员当前所属组织修改和操作日志记录，后续版本再扩展人员历史归属和月度人员快照。

## 七、车间管理员需求

车间管理员负责本车间及下属工区的数据管理。

### 1. 可使用功能

车间管理员可以使用以下功能：

1. 登录系统。
2. 修改本人密码。
3. 查看本车间首页看板。
4. 填写车间本级数据。
5. 查看本车间下属工区数据。
6. 审核下属工区提交的数据。
7. 退回下属工区数据。
8. 查看本车间汇总。
9. 查看本车间统计分析。
10. 导出本车间数据。
11. 查看本车间审核记录。
12. 查看本车间人员信息。

### 2. 数据范围

车间管理员只能查看和管理：

```text
本车间
本车间本级
本车间下属工区
本车间人员
本车间填报数据
本车间审核记录
```

### 3. 多个车间管理员需求

一个车间可以配置多个车间管理员或车间领导账号。

业务规则如下：

1. 多个车间管理员可以绑定同一个车间。
2. 绑定同一车间的管理员都可以查看该车间数据。
3. 绑定同一车间的管理员都可以审核该车间下属工区数据。
4. 绑定同一车间的管理员都可以退回该车间下属工区数据。
5. 绑定同一车间的管理员都可以查看本车间汇总和统计分析。
6. 系统需要记录具体是谁进行了审核、退回、修改、导出等操作。

例如：

```text
贺州通信车间可以有：
车间管理员A
车间管理员B
车间主任账号
车间副主任账号
```

这些账号都可以拥有本车间范围内的管理权限，但不能访问其他车间数据。

### 4. 不允许操作

车间管理员不能：

1. 查看其他车间数据。
2. 修改其他车间数据。
3. 管理段级账号。
4. 管理组织架构。
5. 管理用工项目字典。
6. 锁定月份。
7. 解锁月份。
8. 查看全段操作日志。
9. 查看段级汇总数据。

## 十一、用工项目多级表头分析

工时工分填报是本系统最核心的业务模块。

### 1. 不能把 Excel 表头简单做成固定字段

Excel 表头不是简单的一行字段，而是多级合并表头。

例如培训部分：

```text
培训
├── 适应性培训
│   ├── 局内
│   └── 局外
├── 资格性培训
│   ├── 局内
│   └── 局外
├── 厂培
├── 其他类培训
└── 备注：培训班名称
```

这里面包含三种类型：

```text
分类项
数值项
文本项
```

示例：

| 项目              | 类型   | 是否填写数值 |
| ----------------- | ------ | ------------ |
| 培训              | 分类项 | 否           |
| 适应性培训        | 分类项 | 否           |
| 适应性培训 / 局内 | 数值项 | 是           |
| 适应性培训 / 局外 | 数值项 | 是           |
| 厂培              | 数值项 | 是           |
| 其他类培训        | 数值项 | 是           |
| 备注：培训班名称  | 文本项 | 否，填写文字 |

因此系统不能只让用户选择：

```text
培训
```

而是要选择到最终可填写的具体项目，例如：

```text
培训 / 适应性培训 / 局内
培训 / 适应性培训 / 局外
培训 / 资格性培训 / 局内
培训 / 资格性培训 / 局外
培训 / 厂培
培训 / 其他类培训
```

### 2. 用工项目应设计为树形字典

系统中应把 Excel 表头拆成一棵用工项目树。

例如：

```text
施工
├── Ⅰ级施工
│   ├── 准备时间
│   ├── 往返路程时间
│   ├── 施工时间
│   └── 备注
├── Ⅱ级施工
│   ├── 准备时间
│   ├── 往返路程时间
│   ├── 施工时间
│   └── 备注
└── Ⅲ级施工
    ├── 准备时间
    ├── 往返路程时间
    ├── 施工时间
    └── 备注
```

再例如：

```text
培训
├── 适应性培训
│   ├── 局内
│   └── 局外
├── 资格性培训
│   ├── 局内
│   └── 局外
├── 厂培
├── 其他类培训
└── 备注：培训班名称
```

也就是说：

1. 施工、培训、故障处理等是一级分类。
2. Ⅰ级施工、适应性培训等是中间分类。
3. 准备时间、施工时间、局内、局外、厂培等才是真正可填写的数值项目。
4. 备注、培训班名称等属于文本项。

## 十二、工时工分填报需求

### 1. 填报方式

填报页面采用“人员 + 日期 + 项目明细”方式。

推荐填报流程为：

```text
选择月份
选择人员
选择日期
选择类别：工时 / 工分
选择完整用工项目路径
填写数值
填写备注或文本内容
保存
```

这里的“选择用工项目”必须理解为：

```text
选择完整项目路径，并且只能选择最终可填写项
```

例如：

```text
施工 / Ⅰ级施工 / 准备时间
施工 / Ⅰ级施工 / 施工时间
培训 / 适应性培训 / 局内
培训 / 厂培
故障处理 / 有线 / 室内 / 高铁
```

### 2. 填报页面示例

新增填报时页面可设计为：

```text
月份：2026年6月
人员：张三
日期：2026-06-01
类别：工时

用工项目：
[培训 / 适应性培训 / 局内 ▼]

数值：
[120] 分钟

培训班名称：
[南宁通信段适应性培训班]

备注：
[填写说明]

[保存]
```

如果类别选择“工分”，则单位显示为：

```text
分
```

例如：

```text
类别：工分
用工项目：培训 / 适应性培训 / 局内
数值：2 分
```

### 3. 一个人一天可以填写多条项目明细

例如张三 2026-06-01 当天有多项工作：

| 项目                      | 数值 | 单位 | 备注     |
| ------------------------- | ---- | ---- | -------- |
| 施工 / Ⅰ级施工 / 准备时间 | 60   | 分钟 | 施工准备 |
| 施工 / Ⅰ级施工 / 施工时间 | 180  | 分钟 | 现场施工 |
| 培训 / 适应性培训 / 局内  | 120  | 分钟 | 安全培训 |

页面应支持添加多条明细，而不是一次只能保存一个字段。

推荐页面结构：

```text
选择月份
选择人员
选择日期
选择类别：工时 / 工分

添加明细：
  用工项目：级联选择
  数值：分钟 / 分
  备注：文本
  [添加]

已添加明细：
  项目路径
  数值
  单位
  备注
  操作：编辑 / 删除

合计：
  自动计算总分钟 / 总分

按钮：
  保存草稿
  提交
```

## 十三、工时与工分数据规则

### 1. 工时规则

工时建议按分钟保存。

```text
用户填写：分钟
数据库保存：分钟
页面汇总可显示：分钟或小时
Excel 汇总可按原模板要求转换
```

例如：

```text
120分钟 = 2小时
```

是否需要在汇总表中转为小时，应根据原 Excel 汇总口径处理。

### 2. 工分规则

工分按分值保存，不做分钟转换。

例如：

```text
2分
3.5分
```

### 3. 备注和文本项规则

备注、培训班名称等不参与合计。

例如：

```text
培训 / 备注：培训班名称
```

这类字段属于文本项，应保存为文字，不参与工时、工分汇总。

## 十四、人员管理需求

人员管理用于维护参与工时、工分填报的人员基础信息。填报数据必须关联具体人员。

### 1. 人员基础字段

人员信息应包含以下字段：

```text
姓名
性别
单位
部门（车间）
班组
出生日期
职位名称
聘任专业职务工种
工种
职级分类
状态
```

字段说明：

| 字段             | 说明                               |
| ---------------- | ---------------------------------- |
| 姓名             | 人员姓名                           |
| 性别             | 男 / 女                            |
| 单位             | 所属单位，例如南宁通信段           |
| 部门（车间）     | 所属车间                           |
| 班组             | 所属班组或具体工区、班组名称       |
| 出生日期         | 人员出生日期                       |
| 职位名称         | 当前职位名称                       |
| 聘任专业职务工种 | 专业职务或工种信息                 |
| 工种             | 人员实际工种                       |
| 职级分类         | 人员职级或岗位分类                 |
| 状态             | 人员当前状态，如在岗、调出、停用等 |

### 2. 人员管理功能

段级管理员可以：

1. 新增人员。
2. 编辑人员。
3. 停用人员。
4. 查询人员。
5. 修改人员所属车间。
6. 修改人员所属工区或班组。
7. 查看人员当前状态。
8. 查看人员调动记录。

车间管理员可以：

1. 查看本车间人员。
2. 查看本车间下属工区人员。
3. 是否允许修改本车间人员信息，可由甲方确认。

工区填报员可以：

1. 查看本工区人员。
2. 在填报时选择本工区人员。
3. 不允许维护人员基础信息。

### 3. 人员调动功能

段级管理员可以将人员从一个工区调整到另一个工区，也可以调整人员所属车间或班组。

调动要求：

1. 支持修改人员所属车间。
2. 支持修改人员所属班组或工区。
3. 支持记录调动前组织和调动后组织。
4. 支持记录调动原因。
5. 支持记录操作人和操作时间。
6. 调动后，新填报数据按新的所属组织归属。
7. 历史月份数据原则上保留原所属组织，避免历史数据被错误覆盖。

## 十五、功能需求

### 1. 登录功能

用户通过账号和密码登录系统。系统根据用户角色加载对应菜单和数据范围。第一版不提供用户自主注册功能，所有账号由段级管理员统一创建。

### 2. 修改密码功能

用户可以修改自己的登录密码。首次使用默认密码登录时，系统可以要求用户先修改密码，再进入系统。

### 3. 首页看板功能

首页展示当前月份的填报进度和汇总情况。不同角色看到不同范围的数据：

```text
段级：查看全段数据
车间：查看本车间数据
工区：查看本工区数据
```

### 4. 账号管理功能

段级管理员维护系统账号，包括新增账号、编辑账号、禁用账号、重置密码、分配角色、绑定车间或工区。

账号规则：

```text
工区账号必须绑定具体工区
车间账号必须绑定具体车间
段级账号拥有全段范围
一个车间可以绑定多个车间管理员账号
```

### 5. 组织架构管理功能

段级管理员维护段、车间、工区结构。支持新增车间、新增具体工区、设置车间下属工区、启用或停用组织。

### 6. 人员管理功能

维护填报人员信息，包括姓名、性别、单位、部门（车间）、班组、出生日期、职位名称、聘任专业职务工种、工种、职级分类、状态等。填报数据必须关联具体人员。

### 7. 人员调动功能

段级管理员可以将人员从一个工区调整到另一个工区，也可以调整人员所属车间或班组。

### 8. 用工项目字典功能

维护所有可填报的用工项目。系统根据用工项目字典生成填报选项，并用于 Excel 导出和统计汇总。

该功能必须支持：

```text
多级项目
分类项
数值项
文本项
是否参与合计
Excel 列映射
项目启用 / 停用
项目排序
默认二十多个用工项目初始化
```

### 9. 月度填报管理功能

段级管理员创建填报月份，设置填报开始时间、截止时间、审核时间，并控制月份开启、关闭、锁定和解锁。

### 10. 工区填报功能

工区填报员填写本工区人员每日工时、工分数据。

功能包括：

```text
选择月份
选择人员
选择日期
选择工时或工分
选择完整用工项目路径
填写分钟或分值
填写备注或文本项
保存草稿
编辑草稿
删除草稿
提交本工区数据
查看退回原因
退回后重新提交
```

工区账号只能操作本工区数据。

### 11. 车间本级填报功能

车间管理员填写车间本级人员的工时、工分数据。该功能与工区填报类似，但数据归属于车间本级。

### 12. 车间审核功能

车间管理员审核下属工区提交的数据。

功能包括：

```text
查看下属工区提交状态
查看工区填报明细
查看工区汇总
审核通过
退回修改
填写退回原因
批量审核
提交车间数据到段级
```

同一个车间可以有多个车间管理员，这些管理员均可对本车间下属工区数据进行审核和退回操作。

### 13. 车间汇总功能

车间管理员可以查看本车间汇总数据，包括车间本级和下属工区数据。

支持：

```text
按工区汇总
按人员汇总
按用工项目汇总
按工时汇总
按工分汇总
查看提交状态
查看退回状态
导出本车间数据
```

### 14. 段级汇总功能

段级管理员查看所有车间、所有工区的汇总情况。

支持：

```text
按车间汇总
按工区汇总
按人员汇总
按用工项目汇总
按月份汇总
查看提交状态
导出汇总表
```

### 15. 统计分析功能

系统需要增加统计分析管理模块，用于对工时、工分数据进行多维度分析。

统计分析建议支持以下维度：

```text
按月份统计
按车间统计
按工区统计
按人员统计
按用工项目统计
按工时统计
按工分统计
按填报状态统计
按审核状态统计
```

统计分析页面建议展示：

1. 当前月份总工时。
2. 当前月份总工分。
3. 各车间工时对比。
4. 各车间工分对比。
5. 各工区工时对比。
6. 各工区工分对比。
7. 用工项目占比。
8. 人员工时排名。
9. 人员工分排名。
10. 未提交工区统计。
11. 被退回数据统计。

不同角色看到的统计范围不同：

```text
段级管理员：查看全段统计分析
车间管理员：查看本车间统计分析
工区填报员：查看本工区统计分析
```

### 16. Excel 导出功能

系统支持导出 Excel。

导出范围按权限控制：

```text
工区账号：只能导出本工区
车间账号：只能导出本车间
段级账号：可以导出全段或指定车间、工区
```

导出要求：

1. 工区导出的 Excel 表格需要尽量与原工区模板一致。
2. 导出表格应保留原有多级表头结构。
3. 导出表格应保留原有工作表名称。
4. 导出表格应保留合计列。
5. 导出表格应保留备注列。
6. 导出表格应正确处理工时、工分、文本项。
7. 段级导出时可导出汇总表。
8. 车间导出时可导出本车间汇总表和下属工区表。
9. 工区导出时只导出本工区数据。

### 17. 操作日志功能

系统记录关键操作，包括：

```text

新增
修改
删除
提交
审核
退回
导出
锁定
解锁
人员调动
账号禁用
密码重置
```

日志用于后期追溯数据修改过程。

### 18. 审核记录功能

系统记录工区提交、车间审核、退回修改、车间提交段级、段级锁定等流程记录，便于查看数据流转过程。

审核记录应包含：

```text
操作人
操作时间
操作组织
操作类型
审核结果
退回原因
备注
```

## 二十二、Excel 导出需求重点

## 二十六、系统难点总结

本系统的难点不在普通增删改查，而在以下几点：

1. Excel 多级合并表头需要拆成树形用工项目字典。
2. 默认二十多个用工项目需要完整初始化和维护。
3. 部分项目是数值项，部分项目是文本项，不能混在一起处理。
4. 工时和工分需要分开统计。
5. 工区、车间、段级数据权限必须严格控制。
6. 真实工区名称和 Excel 模板 A-H 工作表之间需要建立映射关系。
7. 一个车间可能存在多个管理员，需要支持多人管理同一车间。
8. 人员可能发生工区或车间调动，需要处理当前归属和历史数据关系。
9. 工区填报、车间审核、段级汇总流程必须形成闭环。
10. 统计分析需要根据不同角色展示不同范围的数据。
11. 工区导出的 Excel 表格需要尽量与原模板一致。
12. 已提交、已审核、已锁定数据的修改规则必须明确。
13. 所有关键操作需要记录日志，方便追溯。

## 二十七、最终需求结论

本系统核心目标是：

> 建立一个按“段—车间—具体工区”三级权限管理的劳动用工工时、工分线上填报系统，实现工区填报、车间审核、段级汇总、统计分析、Excel 导出和全过程数据追溯。

最关键的设计点是：

```text
车间下属工区使用真实工区名称
每个车间下属 3-7 个具体工区
A-H 可作为 Excel 工作表映射，不作为真实工区名称
用工项目不能简单做成固定字段
必须设计成多级树形字典
默认二十多个用工项目需要全部内置
用户填报时选择最终可填写的项目
数值项填写分钟或工分
文本项填写备注、培训班名称等内容
人员信息需包含姓名、性别、单位、部门、班组、出生日期、职位名称、聘任专业职务工种、工种、职级分类、状态
人员可以由段级管理员进行工区或车间调动
一个车间可以配置多个车间管理员或领导账号
系统需要提供统计分析功能
工区导出的 Excel 表格需要尽量和原模板保持一致
```

完成后，系统应具备以下业务闭环能力：

```text
段级建组织、账号、人员、月份、用工项目
        ↓
工区填报工时、工分
        ↓
工区提交
        ↓
车间审核或退回
        ↓
车间提交段级
        ↓
段级汇总和统计分析
        ↓
段级导出 Excel
        ↓
段级锁定月份
```

该闭环完成后，系统即可满足上线试用和甲方演示确认需求。
